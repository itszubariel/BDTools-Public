Node Status

Monitor BDFD node availability, ping, and bot counts across all standard and high-performance nodes. Data refreshes every 5 minutes.

Node Status

Overview GET /node-status GET /node-status/summary GET /node-status/nodes GET /node-status/node/:id GET /node-status/high-performance GET /node-status/history

Overview

The Node Status API provides real-time visibility into the health of BDFD's infrastructure. It exposes per-node metrics (ping, bot count, online/offline status) for both standard and high-performance node pools, plus a rolling history of snapshots for trend analysis.

No auth required Updated every 5 minutes 
// Base URL for all node status endpoints
https://api.bdtools.xyz/node-status
GET /node-status

Returns a full status overview including summary counts and all nodes grouped into standard, high-performance, and offline arrays. This is the primary endpoint for a complete snapshot of the BDFD infrastructure.

No auth required Updated every 5 minutes

GET /node-status/summary

Alias for GET /node-status. Returns the identical full response with summary counts and all node arrays. Useful when you want a semantically explicit URL.

No auth required Updated every 5 minutes

Responses

200 OK 
// Same shape as GET /node-status
{
  "totalBots": 812,
  "onlineNodes": 12,
  "offlineNodes": 1,
  "standardNodeCount": 13,
  "hpNodeCount": 1,
  "updatedAt": "2026-04-30T12:00:00.000Z",
  "nodes": {
    "standard": [ /* node objects */ ],
    "highPerformance": [ /* node objects */ ],
    "offline": [ /* node objects */ ]
  }
}
500 Internal Server Error 
{ "error": "Internal server error" }
GET /node-status/nodes

Returns a flat array of all standard nodes sorted by node ID. Use this when you only need the standard node pool without summary metadata or high-performance nodes.

No auth required Updated every 5 minutes

Responses

200 OK 
[
  {
    "nodeId": 1,
    "bots": 14,
    "pingMs": 2,
    "status": "online"
  },
  {
    "nodeId": 2,
    "bots": 13,
    "pingMs": 1,
    "status": "online"
  }
  // ... sorted by nodeId ascending
]
500 Internal Server Error 
{ "error": "Internal server error" }
GET /node-status/node/:id

Returns the current status of a single standard node by its numeric ID. Replace :id with the node number, e.g. /node-status/node/5.

No auth required Updated every 5 minutes

Path Parameters

Parameter Type Description
id integerrequired The numeric ID of the standard node to retrieve.

Responses

200 OK 
{
  "nodeId": 5,
  "bots": 12,
  "pingMs": 1,
  "status": "online"
}
404 Not Found 
{ "error": "Node not found" }
500 Internal Server Error 
{ "error": "Internal server error" }
GET /node-status/high-performance

Returns a flat array of all high-performance (HP) nodes sorted by node ID. HP nodes typically serve premium bots and have lower ping and higher capacity than standard nodes.

No auth required Updated every 5 minutes

Responses

200 OK 
[
  {
    "nodeId": 1,
    "bots": 15,
    "pingMs": 1,
    "status": "online"
  },
  {
    "nodeId": 2,
    "bots": 12,
    "pingMs": 1,
    "status": "online"
  }
  // ... sorted by nodeId ascending
]
500 Internal Server Error 
{ "error": "Internal server error" }
GET /node-status/history

Returns an array of historical status snapshots in chronological order (oldest first). Each snapshot is recorded every 5 minutes and retained for 7 days. Use the limit query parameter to control how many records are returned. Ideal for building uptime graphs or trend charts.

No auth required Updated every 5 minutes

Query Parameters

Parameter Type Default Description
limit integer 2016 Number of snapshots to return. Maximum is 2016 (7 days at 5-min intervals). Use 288 for the last 24 hours.

Responses

200 OK 
[
  {
    "totalBots": 818,
    "onlineNodes": 13,
    "offlineNodes": 0,
    "standardNodeCount": 13,
    "hpNodeCount": 1,
    "createdAt": "2026-04-30T11:55:00.000Z"
  },
  {
    "totalBots": 812,
    "onlineNodes": 12,
    "offlineNodes": 1,
    "standardNodeCount": 13,
    "hpNodeCount": 1,
    "createdAt": "2026-04-30T12:00:00.000Z"
  }
  // ... sorted oldest → newest (chronological)
]
500 Internal Server Error 
{ "error": "Internal server error" }
